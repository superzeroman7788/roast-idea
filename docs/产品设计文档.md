# ROAST · 点子陪练(Sparring Council)— 产品设计文档

> 版本:2026-06 四站重构完成版 · 本文档描述产品**当前实现形态**(非历史规划)。

---

## 0. 一句话与北极星

**ROAST 是一个白箱、人在环、反共识的 AI 思考陪练台。**

- **北极星:不是给你更准的答案,是帮你想明白。**
- 不替你下结论;把多个 AI 的署名观点、分歧、证据**摊开在桌面上**,由**你**来策展、拍板、收敛。
- 反共识:刻意保留"最硬的反对意见",不让它被多数票或主席悄悄抹平。

适用人群:独立开发者 / 产品人 / 创作者 —— 灵感很碎,需要一个工具把碎片接住、想清楚、压力测试、再变成能动手的交付物。

## 1. 整体架构:四站工作台

一条点子 = 一个 **discussion(工作台)**,在**四个相互独立的能力站**之间流转;站与站之间用 **MD 文档交接**。

```
①搜索 ──→ ②陪练 ──→ ③议会 ──→ ④产出
事实侦察   想清楚    温和/拷问   生图/文/PPT
```

- **顶部步骤条**(topbar + steps)是全局共用的导航,四站点一下即切;做完的站亮绿勾。
- **一个 discussion 贯穿四站**:同一条点子攒下四份产物(证据包 / 方向卡 / 议会观点+收敛 / 交付物),共享同一段对话与证据。
- **交接 = MD**:核心交接物是陪练收口出的**方案文档**(主脑 Claude 读整段对话 + 方向卡 + 点赞/纠偏信号 → 固定分节 spec),它才是交给下游精修的"真正方案";方向卡只是想清楚阶段的薄辅助。其余产出也序列化为规范 MD(`evidenceToMd` / `cardToMd` / `convergedToMd`),经左栏「送到」注入下一站 prompt。白箱:全可读可改可下载。
- **专屏专仪表**:每站有自己的三栏布局与中央仪表,切站只换中央 + 侧栏内容,外壳不动。

### 数据模型(discussion)

| 字段 | 含义 | 产出站 |
|---|---|---|
| `brief` | 原始点子文本 | 起点 |
| `turns[]` | 陪练对话(你 ↔ 协同搭子;带 round/speaker/role + **pinned 点赞** / **corrected+correction 纠偏**) | 陪练 |
| `evidencePack` | 证据包(items + byCategory + sources + failures + **brief 简报**) | 搜索 |
| `relay.card` | 方向卡(DirectionCard,薄,想清楚辅助) | 陪练「理清了」 |
| `solutionDoc` | **方案文档**(主脑收口的厚 spec,固定分节,交下游精修的真正方案) | 陪练「出方案文档」 |
| `viewpoints[]` | 各 AI 署名观点(stance / roleAngle / isHardestKill / verification / round) | 议会 |
| `deliberation` | 审议综述(consensus / contradictions / blindSpots …) | 议会 |
| `humanSignals[]` | 人策展信号(endorse/reject/setAside/pin/reply) | 议会 |
| `converged` | 收敛方案(clarified / addressed / unsilenceable / verdictVote …) | 议会收敛 |
| `artifacts[]` | 交付物(type / provider / content / imagePath / status) | 产出 |
| `userId` | 归属用户(用户体系分租,只看自己的) | 全局 |

## 2. 顶部外壳与导航

- **topbar**:`ROAST · SPARRING COUNCIL · 点子陪练` + `已连接 · N 家`(在线 provider 数)。
- **steps**:四步骤条(① 搜索 → ② 陪练 → ③ 议会 → ④ 产出),`done`(亮绿勾)/ `active`(青色高亮)态;点击 = `switchTab`。右侧 `席位 · N` / `历史 · N` / `WORKSPACE · 一条点子 · 四站流转`。
- **ErrorBoundary** 兜底渲染崩溃(显示"刷新重试",不再整页黑屏);SSE 错误净化(乱码/二进制 → 人话提示)。

## 3. 四站详解

每站三栏:**左 = 上下文/导航 · 中 = 主工作区 + 底部输入条 · 右 = 产出/策展**。视觉沿用 JARVIS HUD 暗色主题(`theme.css`,青/琥珀/绿/红色票 + mono 字体)。

### 3.1 搜索站(事实侦察)`ssBody`

**目的**:给点子找证据落点 —— 竞品 / 需求 / 痛点 / 定价 / 趋势。

- **左 · 侦察维度 RECON**:当前点子 + 动效 mini 雷达 + **维度导航**(全部/竞品/需求/痛点/定价/趋势,带 已扫/待扫 状态点 + 命中数),点击筛中间证据流。
- **中 · 证据流**:**证据卡**(编号 / 维度色标 / 来源色点 / 热度 / **纳入简报**开关 / 标题 / 影响分析 / **可信度数字条**);按选中维度过滤;底部搜索条。
- **右 · 侦察简报 BRIEF**:命中数 + 整体可信度两数字卡 + **关键结论**(LLM 合成的色标要点)+ **侦察建议**(够不够进下一站)+ 送到 陪练/议会。

**引擎(`evidence.mjs`)**:
- **多源并行检索**(channel 适配器,`Promise.allSettled` 兜底不空屏):GitHub、Hacker News、Exa(经 mcporter)、V2EX、**趋势**(HN search_by_date 近期高热)、SearXNG(可选)。
- 中文点子 → `extractKeywordsViaAI`(gpt-4o-mini)抽英文搜索词;中文源用原文查询。
- 去重(按 url)+ 排序(可信度优先 → 信号强度)+ top 12 + Jina 正文增强(top 3)。
- **侦察简报合成**(`synthesizeEvidenceBrief`,对冲调用 ~5-10s):读全部证据 → `{conclusions, confidence, suggestion, categories}`。其中 **categories 按内容重判类目**(LLM + 强规则:应用商店/Show HN/GitHub → 竞品)修正来源粗分,让维度真分布。
- 「重新搜索」改了点子 → 重检索补进**同一讨论**(不新建、不丢对话),新点子也回写 `discussion.brief`。

### 3.2 陪练站(想清楚)`llBody`

**目的**:专注对话把碎片想清楚 → 出方向卡(薄辅助)→ **出方案文档(厚,交下游)**。空场中央是「IDEA · 核心」三脑环绕的 HUD 星图。

- **左 · 对话时间线**:`turns` 扁平列表(角色色票 + 摘要),**新发言自动滚到最新**(贴底才跟随,滚上去看历史不打扰,浮出「↓ 跳到最新」);点开某条进**中央详情视图**(大图 + 上下文上/下条 + 追问此条 + **详情页也有输入框**)。
- **中 · 三脑并列**:**对话搭子**选择(主脑 / 2 脑 / 3 脑,默认主脑=单栏=专注对话);每次发问,选中的 N 个脑**并列成栏**作答,每栏 = 模型 pill + 角色 + 正文 + **「👍 点赞 · 纳入主脑」/「👎 纠偏」**;顶部「理清了 · 出方向卡」。脑过载兜底时卡上标「↪ X 过载 · Y 接棒」。
- **右 · 方向卡 DIRECTION + 方案文档**:上半=主脑方案(点赞的 pinned)+ relayCard 各段(已稳定 / 新角度 / 关键假设 / 需你拍板 / 暂不做,可折叠);下半=**「📄 出方案文档」**入口,出了之后变「✓ 方案文档已出 → ⚖议会压测 / ⚡送到产出」。

**引擎**:
- **对话**:`/respond`(clarify)→ `runDiscussionRound` 各脑并行回应(host/builder 协同角色,非对抗);第一次发送**即时反馈**(busy 早置真),且**起手不检索证据**(`redacted=true`,省 8-15s)。同一轮并行看不到彼此,**跨轮**经 `DISCUSSION SO FAR` 都读得到对方上一条 + 你的赞/纠偏。
- **点赞(👍)/ 纠偏(👎)** —— 一对人在环信号,都广播给全桌:
  - 👍 `pinnedBlock`:把这条标「⭐ 优先照顾」注入后续每个脑 + 出卡/方案文档优先纳入。
  - 👎 `correctionBlock`:把这条标「🚫 已排除方向」注入后续每个脑(别再跑偏)+ 出卡/方案文档当排除项;**同时让这条脑当场带你的纠正重答**(`reanswerTurn`,就地替换);白箱标红 + ↻已纠偏重答 + 可撤销。
- **理清了 → 方向卡**(`runClarifyCard`,薄):Claude 抢跑、~6s 没出补 OpenAI、对冲出 `DirectionCard`。
- **出方案文档**(`runSolutionDoc`,厚):主脑读整段对话 + 方向卡 + 赞/纠偏 → **固定分节 spec**(一句话定位/问题/目标用户/核心方案/P0·P1功能/MVP范围/假设/风险/里程碑/需拍板);存 `solution_doc`,是交下游精修的真正方案。主脑挂了依次兜底。

### 3.3 议会站(温和/拷问)`ycBody`

**目的**:多个 AI 署名开火压力测试,**你来策展**,收敛成方案。

- **左 · 议程 MOTIONS**:受审点子 + **议题**(从议会综述派生:共识→已认领 / 分歧·盲点→待裁)+ 认领/否决/搁置计数;点议题松筛中间观点。
- **中 · 多 AI 审议**:**审议强度**(温和·综述 / 拷问·开火)+ 送进产出收敛;**署名观点卡**(模型 pill / 角色 / 立场〔Ship→主张·Fix→验证·Kill→拷问·Pause→待定〕/ 正文 / flags〔🔴 不可静音的最硬 kill、核查徽章、R3〕+ **✓认领 ✕否决 –搁置 ↳插一句**);底部追问条(= 再审一轮,有策展先确认)。
- **右 · 策展台 CURATION**:认领·进方案 / 否决·不采纳 / 搁置·待定 三段(实时列已策展观点)+ **主席提示**(从分歧派生)+ 收敛成方案→产出 / 导出议会纪要。

**引擎(`runDeliberation`)**:白箱审议链路 —
1. **R1 主脑立靶**(organizer,不当裁判)→ 摆出结构化"靶子"。
2. **R2 反方并行开火**(critics,可配置 persona:投资人/增长/可行性/魔鬼代言人/领域专家…)→ 各出署名观点(stance + evidenceIds + isHardestKill)。
3. **Verifier 核查**:对每条观点的证据引用做事实核查(supported/unsupported/overreach 徽章)。
4. **R3 匿名交叉互驳**(拷问档):反方相互去署名质疑;**自适应停止**(R2 高度一致则跳过)。
5. **主席综述**(chairman,不自由裁决)→ consensus/contradictions/blindSpots。
- **温和(council)⇄ 拷问(roast)**:拷问 = 强制魔鬼 + 不可静音 + R3 交叉。
- **人策展(`/signal`)**:认领=endorse / 否决=reject / 搁置=setAside / 插一句=reply（+ 引擎层保留 pin=不可静音）。
- **收敛(`runConverge`)**:只围绕你**认领/否决**的集合收敛 —— 认领点逐条应对(addressed,带 persona tag)、否决项不采纳、搁置留痕、**最硬 kill 强制写回 unsilenceable**(模型漏了就补;但**被你否决的不强制**)、透明投票(verdictVote tally,仅参考不独断)。

### 3.4 产出站(生图/文/PPT)`ccBody`

**目的**:把想清楚 + 压力测试过的方案,交给某个 AI 变成交付物。

- **左 · 方案源 PLAN**:当前点子 + **完整展示「方案文档 · 主脑收口」**(陪练送来的厚 spec,各模型按各自固定小节精修它)+ 要点速览。`docFor(relay)` 优先送方案文档,没有才退方向卡/收敛;没显式交接也用方案文档当持久方案源。
- **中 · 生成**:**格式五宫格**(文案 / PRD / 设计文档 / 配图 / PPT,每种各有固定小节 schema)+ **交给哪个模型** + ⚡生成;**产物卡流**(图标/类型/模型/正文或图 + **采用** / 改稿 / 换模型重生 / 复制 / 导出 / **配图↓下载**;生成中显骨架)。
- **右 · 产物 OUTPUTS**:已完成/生成中 + 产物清单 + **打包为立项包**(点子 + **方案文档** + 已采用产物 → 一份 MD)+ 导出 DOCX / 复制全部。

**引擎(`runProduce`)**:按 type 分发 —
- 文字类(`buildProducePrompt`):文案 / 一页 PRD / 设计文档 / 代码草稿 / **PPT 演示大纲(## 切页)** —— 各 type 的 prompt 已带固定小节(即"精修 schema")。
- 配图(`generateImage`):OpenAI 生图,落 `/data/artifacts`,经 `/api/artifact/:id/image` 取;产物卡/库支持「↓ 下载图」(同源 `a[download]`)。
- **改稿** = refine / **换模型重生** = 新版 / **采用** = chooseArt(同类只采用一版进立项包)。
- 导出:MD / DOCX / PPTX / 立项包。**全局「我的交付物」库**(顶栏 chip → `GET /api/artifacts`,跨所有点子汇总本用户产物,带 ↓下载 / ↗打开那条点子)。

## 4. 模型层与"对冲"(`providers.mjs`)

- **7 家直连 provider**(BYO-key,统一 OpenAI 兼容 + Anthropic):OpenAI(gpt-4o-mini)、Kimi、DeepSeek、Qwen、Agnes、OpenRouter、Claude。生图用 OpenAI gpt-image-1。
- **按"思考链路位置"分配模型**(非按谁最强):陪练协同搭子默认 Claude / OpenAI / DeepSeek;议会按 persona↔model 解耦可配置。
- **鲁棒性**:`fetchRetry` 重试 429/500/502/503/504/**529**(Anthropic Overloaded);各 seat 失败降级、不静默伪装("反共识"——观点宁可标失败也不编造)。
- **对冲(`hedgedChatJSON` + `staggeredRace`)**:主力两家阶梯竞速 —— Claude 抢跑,grace(~6s)没出就并行补 OpenAI,**第一张有效卡即返回**,两家都挂再依次兜底。`tries:1` 失败即换人 + 超时上限 + maxTokens 给足(中文 JSON 不被截断成非法)。**理清了出卡 + 侦察简报合成**都走它,把"等一家超时"从 ~45s 压到 ~15s。
- **Response Healing**:JSON 解析失败时让模型自我修复一次再解析。
- **传输**:全程 SSE 流式;前端 `streamSSE` + token 取消语义。

## 5. 持久化 · 部署 · 技术栈

- **前端**:React 18 + TypeScript + Vite。单文件 `src/main.tsx`(App + 四站 body)、`discussion.ts`(类型/常量)、`theme.css`(JARVIS HUD 令牌)、`exportDoc.ts`(MD/DOCX/PNG/PPTX 导出)。
- **后端**:Node 单进程(`server/index.mjs`),既托管前端 `dist/` 又提供 `/api` + SSE。`providers.mjs` / `evidence.mjs` / `prompts.mjs` / `db.mjs`。
- **数据库**:**libSQL(异步)**。本地 `file://` 持久;线上 **Turso**(`TURSO_DATABASE_URL`)云端持久 —— 免费层休眠/重启不丢历史。**铁律:db 全函数 async,调用必须 await**(漏 await → 返回 Promise 当数据用 → 崩,曾导致线上黑屏)。
- **部署**:Render 免费层(Docker 单进程,`render.yaml` Blueprint,push master 自动部署)。免费层 15min 闲置休眠 → 冷启动 ~30-60s;已加 **DB 启动预热** + **keepalive**(`.github/workflows/keepalive.yml` 兜底;主力建议外部 cron-job.org 每 10min ping `/api/status`)。
- **隐私**:`ROAST_PERSIST_BRIEF=0` 可关闭存点子正文。

### 用户体系(邮箱登录 + 邀请制 + 分租)

- **登录 = 白名单直登(默认)**:输邮箱 → 在邀请名单内即直接下发 **httpOnly 签名会话 cookie**(HMAC-SHA256 自签 JWT,零依赖,30 天)进台;非名单 → 403。**魔法链接验证机制保留**(`ROAST_DIRECT_LOGIN=0` 切回,大范围内测再开:一次性 token DB 存哈希 15min → 点链接 → cookie;Resend 可选,无则链接打日志兜底)。旧 `ROAST_ACCESS_PASSWORD` 密码门已被取代。
- **邀请名单**:`ROAST_OWNER_EMAIL`(站长,永远在)+ `ROAST_ALLOWED_EMAILS`(逗号分隔)。仓库公开,**邮箱只放私密 env**。
- **分租**:`discussions.user_id`;`list/getDiscussion`、交付物库全按 `user_id` 过滤 + 取单条校验归属(防越权)。启动把历史无主讨论迁给站长。`users` / `magic_tokens` 表。
- **会话端点**:`/api/auth/request`(直登或发链接)、`/api/auth/verify`、`/api/me`、`/api/auth/logout`;鉴权门挡未登录的 `/api/*`(status/auth/me 除外)。

## 6. 贯穿全局的设计决策

| 决策 | 为什么 |
|---|---|
| **反共识** | 最硬的 kill 强制写回 unsilenceable、各 seat 失败不伪装、否决/搁置都留痕 —— 不让 AI 用"综合"把尖锐意见磨平。 |
| **白箱 + 人在环** | 每步产出可读可改、交接走 MD、策展由人做;AI 提问/摆分歧,**人拍板**。 |
| **四站独立 + MD 交接** | 贴用户心智(搜索→想清楚→压测→产出),模块化;一个工作台攒一条点子的四份产物。 |
| **专屏专仪表** | 每站三栏 + 专属中央(搜索=雷达/陪练=三脑并列/议会=观点卡/产出=产物流);切站只换内容。 |
| **对冲提速** | 主力模型(尤其 Claude)常过载,顺序兜底要等超时;阶梯竞速让"理清了/简报"稳定 ~15s。 |
| **陪练惰性起手** | 首发不检索证据(redacted),~34ms 建讨论;证据交给搜索站,不拖慢对话。 |
| **即时反馈** | 点发送即置 busy + 乐观气泡,消除"死按钮"窗口(冷启动/检索期也有反馈)。 |

## 7. 现状与已知小瑕疵

**四站(搜索/陪练/议会/产出)均已是按 claude.ai/design 稿重建的新设计**,经一轮全面审计 + 对抗复核,修掉 14 个 bug/体验问题(含 3 个高危:理清了误清议会、配图裂图、否决空操作)。

**已知待办(不影响主流程)**:
- copy(文案)模式下证据类目错配(GitHub/HN/趋势写死 idea 类目)—— 目前 UI 无切 copy 模式入口,启用前再修。
- 搜索维度"待扫/已扫"按命中数推断、陪练协同搭子沿用设计稿"反方"标签、detailId 跨站不清等小瑕疵。
- 旧 `centerCol`/`rightCol`/`conversationCol`/`composerBar` 已无 tab 走(死代码,无 noUnusedLocals,留着无害,可后续清)。

**议会可靠性调优(2026-06,curl 抓 SSE 实测定位)**:
- **主席综述曾被静默截断**:综述字段多(共识/矛盾/盲点/独到),默认 `maxTokens=1100` 把 JSON 切成非法 `Unterminated string` → 整段综述丢失(`deliberation` 恒空)+ heal 重试白等 45s。修:主席 `maxTokens 2600`/超时 60s、立靶 1800、核查 1600 —— 不再截断,综述稳定落地。**经验:长结构化 JSON 的 seat 必须按字段量给足 token,否则"跑完却没结论"。**
- **反方无兜底**:某反方模型 429/超时直接掉一个署名视角,极端时"卡在只有立靶 1 条"。修:反方加错峰兜底池(不在主阵容的已配置模型)+ `tries:1` 快速失败再换人 —— 某席挂了别家顶上出该视角,不整丢。
- **核查→主席串行慢**:两段各 ~30-45s 叠加 → 总耗时 ~150s。修:核查与主席**并发跑**(综述只读观点正文、不依赖核查徽章)。综合后总耗时 ~85s,且综述/视角不再丢。残余变数是单模型偶发慢(免费层多模型链路固有抖动),由议会分阶段进度条(立靶→反方→核查→主席)显式暴露卡点。

**本轮新增(2026-06,均已实测端到端)**:用户体系(邮箱直登 + 邀请制 + 分租)、陪练 **👍点赞 / 👎纠偏**(纠偏=当场重答 + 广播全桌)、**方案文档**(主脑收口的厚 spec,取代薄方向卡当下游交接物)、**全局交付物库**(跨点子汇总 + 下载)、议会可靠性调优(上)。

**后续方向**:每人 BYO 自带密钥(现共用站长额度)、图片接对象存储、产出各 type 升级为更严的 JSON schema。

## 8. 自动档(⚡ Auto-Pilot)· 第零站加速器 —— 设计方案(规划中,未实现)

> 状态:**已设计、未实现**。原 spec 在 codex-work 仓库 `docs/autopilot-v2-spec.md`,原型为一张整屏 SVG 静态稿(`自动档·Auto-Pilot.html`)。本节把该方案并入主文档存档;落地前应先做下方 Phase 0 小验证。

**定位**:四站的「**第零站 / 加速器**」。模糊点子进 → 自动并行跑 搜索+陪练+议会 的核心动作 → 输出**结构化 MD 粗稿 + 待拷问清单** → 用户手动注入任意站继续细化。**前门不替代**:产出是草稿,收尾把人推进陪练/议会;成功标准 = 用户愿带稿进站打磨,不是用完就走。
- **入口**:步骤条新增「⚡ 自动档」;四站右栏提供反向入口(任意站可跳入)。**独立页三栏**:左=进度摘要 + 收敛度趋势图 / 中=workflow 主舞台 / 右=MD 粗稿预览 + 分发控件。自适应画框 + ≥12px 字 + 弱扫描线。

**Agent 拓扑(复用已上线引擎,不另造)**:导演 Director ×1(调度 / 广播任务单 / 轮末评估 / 终止判断,= 主脑 + `hedgedChatJSON`)+ 产出 Agent ×3(① `direction`+`artifacts_hint` / ② `open_questions[]` / ③ 证据引用+分歧标注,= relay.card / runDeliberation 反方)+ 提问 Agent 0–1(盲点,**优先合进导演轮末省一次调用**)。**角色↔模型解耦**,跨厂商出多样性。

**MD 输出 Schema(三层)**:① 强约束(机器读、注入校验用,四字段必填):`brief_original` / `direction`(一句话方向)/ `open_questions[]`(自动注议会)/ `artifacts_hint`(建议产出类型);② 半约束:各 agent 署名观点卡(角色/立场/证据/分歧,结构固定内容自由);③ 自由:原始对话时间线(白箱回溯)。

**Workflow 五步**:① 导演第一屏(SSE 流式解读点子 + 任务预告,**用户可此刻打断/纠正**)→ ② 导演广播结构化任务单(JSON,左栏:字段状态 + 本轮重点 + 上轮未解分歧 + 本轮透镜)→ ③ 三产出 agent 并行(`Promise.allSettled`,单挂不雪崩)→ ④ 导演轮末结构化评估(`schema_completeness / spec_satisfaction(1-5) / open_issues / convergence_score / stop_recommendation / blind_spots`)→ ⑤ **轮间人工插话(半自动)**:暂停亮总结 + 「插一句」框(人喂新角度=最强反熵),选 续跑 / 插一句 / 够了收草稿。

**三层终止(轮次自适应,非写死)**:规则层(四强约束字段全非空,前端毫秒级、零幻觉)→ LLM 层(规则全过后导演判 SPEC 满足度,省无效推理)→ 硬截断层(30min / **明确 token·$ 上限/run**,最外保险丝;截断时输出收敛分最高的"最佳轮")。

**反熵机制(防复读 / 持续拓宽)**:`convergence_score`(100% 规则、不碰 LLM:每轮四强字段字符级 diff,平均变化率 <5% → 判"开始复读"→ **触发干预**,左栏画收敛趋势折线)。干预分常规/应急控成本:
- **常规(每轮、近零延迟)**:B 换透镜(固定库轮换:pre-mortem / 反演 / 跨行业类比 / 放大·缩小 / 二阶效应 / 用户视角)+ F 盲点驱动(导演每轮标"谁都没提的盲点",下轮专攻未探索区)。
- **应急(仅收敛触发,非常规开销)**:A 定向检索(用 blind_spots/open_issues 当 query 查本讨论证据库)+ C embedding 去重(已说观点账本,太像的打回重生)。E 动态招募 persona 后置(极端平台期才上)。
- 原理:复读=同锅汤反复搅;拓宽=加新食材(A 真证据)+ 换尝法(B 透镜)+ 舀掉重复(C);**最强一味是人**(轮间插话)。

**证据库(MVP 先轻,别上重向量库)**:per-discussion 内存 embedding + 余弦相似度即可支撑 A/C(几十条证据),验证反熵有效再升正式向量库。复用搜索站真证据(`evidence.mjs`),搜索跑完异步写入。**冷启动**:第 1 轮强制禁 A+C(库未就绪),证据 agent 无真证据时标"待检索/假设"、不给可信度、**绝不编造**;第 2 轮起才引真证据。

**分发与注入(快照保命)**:右栏「注入 陪练/议会/产出」→ 校验目标站兼容(陪练要 `direction` 非空;议会要 `open_questions≥1` / 观点含≥1 Kill)→ 不兼容红条列缺失字段、不跳;兼容则**先写快照(`*_auto_backup`,带 `{source:'auto_pilot', roundIndex}`)→ 覆写目标站字段 → 跳转 + 目标站显示"有自动档备份·一键还原"**,快照失败整体回滚、绝不裸覆写。站间连接全人工触发。

**白箱保障**:导演任务单/评估报告全展示在左栏、中栏 agent 流式分角色、右栏 MD 实时更新、任意时刻可打断/轮间插话、`convergence_score` 旁附人话解释。

**给实现者(对齐已上线)**:Auto-Pilot = **loop 控制器**,按轮调已上线引擎函数、不复制逻辑;读写 discussion 既有字段(direction→relay.card / 证据→evidencePack / 观点→viewpoints)。绝不编造证据;注入先快照后覆写失败回滚;收敛分前端算、字段门优先于 diff;全程 SSE + `Promise.allSettled` + 失败标记不伪装。

**落地分期(建议 —— 先验证核心假设再投入,非 spec 原文)**:
- **Phase 0(~10% 工作量)**:1 导演 + **1** agent、跑 2 轮、无反熵/无 embedding/无花哨 UI,出 4 字段 MD + 轮间插话 + 注入某站。**只验一件事:自动稿到底有没有用。**
- **Phase 1**:全 loop(3 agent、分层终止、快照注入、三栏 UI)。
- **Phase 2(门控在"真观察到复读")**:反熵 B/F 先,A/C 后;embedding 证据库。
- **Phase 3**:E 动态招募、正式向量库。
- **动手前待敲定**:① 自动稿"比手动跑站强"的底气;② 收敛判定是否初期主要靠人(字符 diff 偏脆);③ 入口定位(加速器排最前 vs 独立元工具);④ 每 run token/$ 硬上限。
