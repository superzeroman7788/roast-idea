# ROAST · 点子陪练(Sparring Council)— 产品设计文档

> 版本:2026-06 四站重构完成版 · 本文档描述产品**当前实现形态**(非历史规划)。

---

## 0. 一句话与北极星

**ROAST 是一个白箱、人在环、反共识的 AI 思考陪练台。**

- **北极星:不是给你更准的答案,是帮你想明白。**
- 不替你下结论;把多个 AI 的署名观点、分歧、证据**摊开在桌面上**,由**你**来策展、拍板、收敛。
- 反共识:刻意保留"最硬的反对意见",不让它被多数票或主席悄悄抹平。

适用人群:独立开发者 / 产品人 / 创作者 —— 灵感很碎,需要一个工具把碎片接住、想清楚、压力测试、再变成能动手的交付物。

## 1. 整体架构:四站工作台

一条点子 = 一个 **discussion(工作台)**,在**四个相互独立的能力站**之间流转;站与站之间用 **MD 文档交接**。

```
①搜索 ──→ ②陪练 ──→ ③议会 ──→ ④产出
事实侦察   想清楚    温和/拷问   生图/文/PPT
```

- **顶部步骤条**(topbar + steps)是全局共用的导航,四站点一下即切;做完的站亮绿勾。
- **一个 discussion 贯穿四站**:同一条点子攒下四份产物(证据包 / 方向卡 / 议会观点+收敛 / 交付物),共享同一段对话与证据。
- **交接 = MD**:每站把自己的产出序列化成规范 Markdown(`evidenceToMd` / `cardToMd` / `convergedToMd`),通过左栏「送到」注入下一站的 prompt(白箱:用户可读可改)。
- **专屏专仪表**:每站有自己的三栏布局与中央仪表,切站只换中央 + 侧栏内容,外壳不动。

### 数据模型(discussion)

| 字段 | 含义 | 产出站 |
|---|---|---|
| `brief` | 原始点子文本 | 起点 |
| `turns[]` | 陪练对话(你 ↔ 协同搭子,带 round/speaker/role/pinned) | 陪练 |
| `evidencePack` | 证据包(items + byCategory + sources + failures + **brief 简报**) | 搜索 |
| `relay.card` | 方向卡(DirectionCard) | 陪练「理清了」 |
| `viewpoints[]` | 各 AI 署名观点(stance / roleAngle / isHardestKill / verification / round) | 议会 |
| `deliberation` | 审议综述(consensus / contradictions / blindSpots …) | 议会 |
| `humanSignals[]` | 人策展信号(endorse/reject/setAside/pin/reply) | 议会 |
| `converged` | 收敛方案(clarified / addressed / unsilenceable / verdictVote …) | 议会收敛 |
| `artifacts[]` | 交付物(type / provider / content / imagePath / status) | 产出 |

## 2. 顶部外壳与导航

- **topbar**:`ROAST · SPARRING COUNCIL · 点子陪练` + `已连接 · N 家`(在线 provider 数)。
- **steps**:四步骤条(① 搜索 → ② 陪练 → ③ 议会 → ④ 产出),`done`(亮绿勾)/ `active`(青色高亮)态;点击 = `switchTab`。右侧 `席位 · N` / `历史 · N` / `WORKSPACE · 一条点子 · 四站流转`。
- **ErrorBoundary** 兜底渲染崩溃(显示"刷新重试",不再整页黑屏);SSE 错误净化(乱码/二进制 → 人话提示)。

## 3. 四站详解

每站三栏:**左 = 上下文/导航 · 中 = 主工作区 + 底部输入条 · 右 = 产出/策展**。视觉沿用 JARVIS HUD 暗色主题(`theme.css`,青/琥珀/绿/红色票 + mono 字体)。

### 3.1 搜索站(事实侦察)`ssBody`

**目的**:给点子找证据落点 —— 竞品 / 需求 / 痛点 / 定价 / 趋势。

- **左 · 侦察维度 RECON**:当前点子 + 动效 mini 雷达 + **维度导航**(全部/竞品/需求/痛点/定价/趋势,带 已扫/待扫 状态点 + 命中数),点击筛中间证据流。
- **中 · 证据流**:**证据卡**(编号 / 维度色标 / 来源色点 / 热度 / **纳入简报**开关 / 标题 / 影响分析 / **可信度数字条**);按选中维度过滤;底部搜索条。
- **右 · 侦察简报 BRIEF**:命中数 + 整体可信度两数字卡 + **关键结论**(LLM 合成的色标要点)+ **侦察建议**(够不够进下一站)+ 送到 陪练/议会。

**引擎(`evidence.mjs`)**:
- **多源并行检索**(channel 适配器,`Promise.allSettled` 兜底不空屏):GitHub、Hacker News、Exa(经 mcporter)、V2EX、**趋势**(HN search_by_date 近期高热)、SearXNG(可选)。
- 中文点子 → `extractKeywordsViaAI`(gpt-4o-mini)抽英文搜索词;中文源用原文查询。
- 去重(按 url)+ 排序(可信度优先 → 信号强度)+ top 12 + Jina 正文增强(top 3)。
- **侦察简报合成**(`synthesizeEvidenceBrief`,对冲调用 ~5-10s):读全部证据 → `{conclusions, confidence, suggestion, categories}`。其中 **categories 按内容重判类目**(LLM + 强规则:应用商店/Show HN/GitHub → 竞品)修正来源粗分,让维度真分布。
- 「重新搜索」改了点子 → 重检索补进**同一讨论**(不新建、不丢对话),新点子也回写 `discussion.brief`。

### 3.2 陪练站(想清楚)`llBody`

**目的**:专注对话把碎片想清楚 → 召主脑收口出方向卡。

- **左 · 对话时间线**:`turns` 扁平列表(角色色票 + 摘要),**新发言自动滚到最新**(贴底才跟随,滚上去看历史不打扰,浮出「↓ 跳到最新」);点开某条进**中央详情视图**(大图 + 上下文上/下条 + 追问此条 + **详情页也有输入框**)。
- **中 · 三脑并列**:**对话搭子**选择(主脑 / 2 脑 / 3 脑,默认主脑=单栏=专注对话);每次发问,选中的 N 个脑**并列成栏**作答,每栏 = 模型 pill + 角色 + 正文 +「点赞 · 纳入主脑」;顶部「理清了 · 出方向卡」。
- **右 · 方向卡 DIRECTION**:**主脑方案**(点赞收进来的 pinned 观点)+ relayCard 各段(已稳定 / 接力新角度 / 关键假设 / 需你拍板 / 暂不做),**分段可折叠 + 整卡收起/展开**。

**引擎**:
- **对话**:`/respond`(clarify)→ `runDiscussionRound` 各脑并行回应(host/builder 协同角色,非对抗);第一次发送**即时反馈**(busy 早置真),且**起手不检索证据**(`redacted=true`,省 8-15s;证据交给搜索站按需补)。
- **点赞(👍)**:pinned 落库,主脑回应 + 出卡时优先照顾(`pinnedBlock` 注入 prompt)。
- **理清了 → 单脑收口 + 对冲**(`runClarifyCard`):**Claude 抢跑,~6s 没出卡就并行补 OpenAI**,谁先出有效卡谁赢(偏好 Claude),两家都挂再依次兜底 —— 读整段对话一次出方向卡(`DirectionCard`)。替代了原来"每条反馈重跑 6 棒接力"(太慢/太费)。

### 3.3 议会站(温和/拷问)`ycBody`

**目的**:多个 AI 署名开火压力测试,**你来策展**,收敛成方案。

- **左 · 议程 MOTIONS**:受审点子 + **议题**(从议会综述派生:共识→已认领 / 分歧·盲点→待裁)+ 认领/否决/搁置计数;点议题松筛中间观点。
- **中 · 多 AI 审议**:**审议强度**(温和·综述 / 拷问·开火)+ 送进产出收敛;**署名观点卡**(模型 pill / 角色 / 立场〔Ship→主张·Fix→验证·Kill→拷问·Pause→待定〕/ 正文 / flags〔🔴 不可静音的最硬 kill、核查徽章、R3〕+ **✓认领 ✕否决 –搁置 ↳插一句**);底部追问条(= 再审一轮,有策展先确认)。
- **右 · 策展台 CURATION**:认领·进方案 / 否决·不采纳 / 搁置·待定 三段(实时列已策展观点)+ **主席提示**(从分歧派生)+ 收敛成方案→产出 / 导出议会纪要。

**引擎(`runDeliberation`)**:白箱审议链路 —
1. **R1 主脑立靶**(organizer,不当裁判)→ 摆出结构化"靶子"。
2. **R2 反方并行开火**(critics,可配置 persona:投资人/增长/可行性/魔鬼代言人/领域专家…)→ 各出署名观点(stance + evidenceIds + isHardestKill)。
3. **Verifier 核查**:对每条观点的证据引用做事实核查(supported/unsupported/overreach 徽章)。
4. **R3 匿名交叉互驳**(拷问档):反方相互去署名质疑;**自适应停止**(R2 高度一致则跳过)。
5. **主席综述**(chairman,不自由裁决)→ consensus/contradictions/blindSpots。
- **温和(council)⇄ 拷问(roast)**:拷问 = 强制魔鬼 + 不可静音 + R3 交叉。
- **人策展(`/signal`)**:认领=endorse / 否决=reject / 搁置=setAside / 插一句=reply（+ 引擎层保留 pin=不可静音）。
- **收敛(`runConverge`)**:只围绕你**认领/否决**的集合收敛 —— 认领点逐条应对(addressed,带 persona tag)、否决项不采纳、搁置留痕、**最硬 kill 强制写回 unsilenceable**(模型漏了就补;但**被你否决的不强制**)、透明投票(verdictVote tally,仅参考不独断)。

### 3.4 产出站(生图/文/PPT)`ccBody`

**目的**:把想清楚 + 压力测试过的方案,交给某个 AI 变成交付物。

- **左 · 方案源 PLAN**:当前点子 +「喂给 AI 的要点」(从陪练方向卡 + 议会收敛派生,标 来自陪练/议会)。
- **中 · 生成**:**格式五宫格**(文案 / PRD / 设计文档 / 配图 / PPT)+ **交给哪个模型**(真实已配置 provider 列表)+ ⚡生成;**产物卡流**(图标/类型/模型/正文或图 + **采用** / 改稿 / 换模型重生 / 复制 / 导出;生成中显骨架);底部补充要求条。
- **右 · 产物 OUTPUTS**:已完成/生成中 + 产物清单 + 下一步建议 + **打包为立项包**(合并点子+方案源+已采用产物成一份 MD)+ 导出 DOCX / 复制全部。

**引擎(`runProduce`)**:按 type 分发 —
- 文字类(`buildProducePrompt`):文案 / 一页 PRD / 设计文档 / 代码草稿 / **PPT 演示大纲(## 切页)**。
- 配图(`generateImage`):OpenAI 生图,落 `/data/artifacts`,经 `/api/artifact/:id/image` 取。
- **改稿** = refine(读旧版改)/ **换模型重生** = 新版(另一家从头出);**采用** = chooseArt(同类只采用一版,进立项包)。
- 导出:MD / DOCX / PPTX(`exportPptx` 按 `## ` 切片)/ 立项包(合并 MD)。

## 4. 模型层与"对冲"(`providers.mjs`)

- **7 家直连 provider**(BYO-key,统一 OpenAI 兼容 + Anthropic):OpenAI(gpt-4o-mini)、Kimi、DeepSeek、Qwen、Agnes、OpenRouter、Claude。生图用 OpenAI gpt-image-1。
- **按"思考链路位置"分配模型**(非按谁最强):陪练协同搭子默认 Claude / OpenAI / DeepSeek;议会按 persona↔model 解耦可配置。
- **鲁棒性**:`fetchRetry` 重试 429/500/502/503/504/**529**(Anthropic Overloaded);各 seat 失败降级、不静默伪装("反共识"——观点宁可标失败也不编造)。
- **对冲(`hedgedChatJSON` + `staggeredRace`)**:主力两家阶梯竞速 —— Claude 抢跑,grace(~6s)没出就并行补 OpenAI,**第一张有效卡即返回**,两家都挂再依次兜底。`tries:1` 失败即换人 + 超时上限 + maxTokens 给足(中文 JSON 不被截断成非法)。**理清了出卡 + 侦察简报合成**都走它,把"等一家超时"从 ~45s 压到 ~15s。
- **Response Healing**:JSON 解析失败时让模型自我修复一次再解析。
- **传输**:全程 SSE 流式;前端 `streamSSE` + token 取消语义。

## 5. 持久化 · 部署 · 技术栈

- **前端**:React 18 + TypeScript + Vite。单文件 `src/main.tsx`(App + 四站 body)、`discussion.ts`(类型/常量)、`theme.css`(JARVIS HUD 令牌)、`exportDoc.ts`(MD/DOCX/PNG/PPTX 导出)。
- **后端**:Node 单进程(`server/index.mjs`),既托管前端 `dist/` 又提供 `/api` + SSE。`providers.mjs` / `evidence.mjs` / `prompts.mjs` / `db.mjs`。
- **数据库**:**libSQL(异步)**。本地 `file://` 持久;线上 **Turso**(`TURSO_DATABASE_URL`)云端持久 —— 免费层休眠/重启不丢历史。**铁律:db 全函数 async,调用必须 await**(漏 await → 返回 Promise 当数据用 → 崩,曾导致线上黑屏)。
- **部署**:Render 免费层(Docker 单进程)。免费层 15min 闲置休眠 → 冷启动 ~30-60s;已加 **DB 启动预热** + **keepalive**(`.github/workflows/keepalive.yml` 兜底;主力建议外部 cron-job.org 每 10min ping `/api/status`)。
- **分发门**:`ROAST_ACCESS_PASSWORD` 软密码门(不进前端包/不进 git)。
- **隐私**:`ROAST_PERSIST_BRIEF=0` 可关闭存点子正文。

## 6. 贯穿全局的设计决策

| 决策 | 为什么 |
|---|---|
| **反共识** | 最硬的 kill 强制写回 unsilenceable、各 seat 失败不伪装、否决/搁置都留痕 —— 不让 AI 用"综合"把尖锐意见磨平。 |
| **白箱 + 人在环** | 每步产出可读可改、交接走 MD、策展由人做;AI 提问/摆分歧,**人拍板**。 |
| **四站独立 + MD 交接** | 贴用户心智(搜索→想清楚→压测→产出),模块化;一个工作台攒一条点子的四份产物。 |
| **专屏专仪表** | 每站三栏 + 专属中央(搜索=雷达/陪练=三脑并列/议会=观点卡/产出=产物流);切站只换内容。 |
| **对冲提速** | 主力模型(尤其 Claude)常过载,顺序兜底要等超时;阶梯竞速让"理清了/简报"稳定 ~15s。 |
| **陪练惰性起手** | 首发不检索证据(redacted),~34ms 建讨论;证据交给搜索站,不拖慢对话。 |
| **即时反馈** | 点发送即置 busy + 乐观气泡,消除"死按钮"窗口(冷启动/检索期也有反馈)。 |

## 7. 现状与已知小瑕疵

**四站(搜索/陪练/议会/产出)均已是按 claude.ai/design 稿重建的新设计**,经一轮全面审计 + 对抗复核,修掉 14 个 bug/体验问题(含 3 个高危:理清了误清议会、配图裂图、否决空操作)。

**已知待办(不影响主流程)**:
- copy(文案)模式下证据类目错配(GitHub/HN/趋势写死 idea 类目)—— 目前 UI 无切 copy 模式入口,启用前再修。
- 搜索维度"待扫/已扫"按命中数推断、陪练协同搭子沿用设计稿"反方"标签、detailId 跨站不清等小瑕疵。
- 旧 `centerCol`/`rightCol`/`conversationCol`/`composerBar` 已无 tab 走(死代码,无 noUnusedLocals,留着无害,可后续清)。

**议会可靠性调优(2026-06,curl 抓 SSE 实测定位)**:
- **主席综述曾被静默截断**:综述字段多(共识/矛盾/盲点/独到),默认 `maxTokens=1100` 把 JSON 切成非法 `Unterminated string` → 整段综述丢失(`deliberation` 恒空)+ heal 重试白等 45s。修:主席 `maxTokens 2600`/超时 60s、立靶 1800、核查 1600 —— 不再截断,综述稳定落地。**经验:长结构化 JSON 的 seat 必须按字段量给足 token,否则"跑完却没结论"。**
- **反方无兜底**:某反方模型 429/超时直接掉一个署名视角,极端时"卡在只有立靶 1 条"。修:反方加错峰兜底池(不在主阵容的已配置模型)+ `tries:1` 快速失败再换人 —— 某席挂了别家顶上出该视角,不整丢。
- **核查→主席串行慢**:两段各 ~30-45s 叠加 → 总耗时 ~150s。修:核查与主席**并发跑**(综述只读观点正文、不依赖核查徽章)。综合后总耗时 ~85s,且综述/视角不再丢。残余变数是单模型偶发慢(免费层多模型链路固有抖动),由议会分阶段进度条(立靶→反方→核查→主席)显式暴露卡点。

**后续方向**:**用户体系(历史按 user_id 分租 + 鉴权 + 每人 BYO 密钥)**、图片接对象存储、Tauri 2 封装 Mac 版(Keychain 存密钥)。
