# Roast My Idea — UI 设计规格（JARVIS / 钢铁侠 HUD 风）

> 配套高保真原型：`docs/ui-mockup-jarvis.html`（浏览器直接打开）
> 给 Claude Code：照这份令牌和组件实现 React + SVG/Canvas，落在 PRD 第 9 节布局上。

---

## 1. 设计基调
深空蓝底 + 青色辉光的 HUD 仪表盘。像贾维斯的全息界面：薄环、刻度、十字准星、四角括线、等宽遥测、克制留白、暗。**辉光是这套的灵魂**（text-shadow / box-shadow / SVG `drop-shadow`），但只给"活的"元素发光（活跃节点、主按钮、状态点），不要满屏发光。

## 3. 布局（三栏 + 顶 chrome + 底输入，对应 PRD 9.2）
- **顶 chrome**：macOS 红黄绿灯 + 居中等宽标题 `ROAST · DECISION COUNCIL` + 右侧"已连接"绿点。
- **左栏 248px**：输入处理器——反应堆 orb（同心环 + 旋转虚线环 + 发光核心）、迷你统计（EVIDENCE / SOURCES / MODELS / CITATIONS，等宽，证据用琥珀、失败用红）、模式切换 IDEA/COPY、底部 VERDICT 卡（绿色发光）。
- **中央**：阶段条（检索›整理›对线›综合，当前步青色发光）+ **实时议会图谱**（见第 4 节）+ 左下图例 + 右下图谱控制。
- **右栏 232px**：遥测——节点/连线/分歧/引用有效率/TOK·S/耗时（等宽，语义着色）+ 流水线 TICK（done 青点、now 琥珀闪烁）+ 底部置信度弧表（标"主观"）。
- **底栏 64px**：青色 `›` 前缀的输入框（粘贴点子，**纯文本无语音**）+ 琥珀发光 `RUN COUNCIL` 按钮。
- 画面四角加青色括线 `corner`，叠扫描线 + 径向暗角。

---

## 4. 核心：实时议会图谱（必做，PRD 9.3 的视觉实现）
- **同心 HUD 环**：3 层静态环 + 2 层旋转虚线环（`spin` 26s / 反向 40s）+ 四向刻度 + 十字准星。
- **节点**：中心=点子核（青、`drop-shadow` 16px、呼吸动画 `core`）；模型=青色描边圆（发言时 `pulse`）；证据=琥珀小圆（`drop-shadow`）；魔鬼代言人=红色描边。
- **连线**：模型→证据=青（引用）；证据→点子=琥珀；模型↔模型=红色虚线（分歧）。
- **标签**：等宽 9–10px，模型名 + 其 stance（KILL 红 / FIX 琥珀 / SHIP 绿）。
- **实现**：React + SVG（节点少时）或 Canvas/WebGL + `react-force-graph`（节点多时）；辉光用 SVG `filter: drop-shadow` 或 CSS。数据驱动：节点/连线来自真实 run（P1 就接真数据，禁止假图）。

---

## 5. 动效（克制，全 CSS / requestAnimationFrame）
- 旋转虚线环 `spin`（26s/40s 线性循环）。
- 核心与活跃节点 `core` / `pulse` 呼吸（2.4–3s）。
- 状态点 / now-tick `blink`（1.4–1.6s）。
- 流式：阶段 B 模型节点**依次点亮**、引用证据时连线**逐条亮起**——把"看 AI 带证据对线"做成实时动画（这是留存与分享点）。
- 性能：辉光用 `drop-shadow` 限量；避免大面积 blur；动画用 transform/opacity。

---

## 6. 状态映射
`idle`（环慢转、核心暗）→ `searching`（证据节点逐个点亮、检索 tick 走）→ `evidence-ready`→ `debating`（模型依次发光、连线亮、红色分歧边出现）→ `validating`（无效引用标红/淡出）→ `verdict`（左下 VERDICT 卡点亮、弧表填充）。`simulated`（<2 真实模型）→ 全局降饱和 + 标"参与不足，非完整议会"。

---

## 7. 实现要点（给 Claude Code）
- 技术：Tauri + React + TS（PRD 第 5 节）。图谱单独组件 `<CouncilGraph data={run}/>`。
- 主题令牌存 CSS 变量，集中一处。
- 辉光/暗底是产品要的，不要套通用浅色组件库；自绘这套 HUD。
- 可访问性：辉光别牺牲对比度，正文 ≥ `#dcecf6`；给纯图标按钮加 aria-label。
- **铁律**：图谱永远渲染真实 run 的节点/连线，不允许写死假数据当展示（参考原型里的数字是示例，实现要接真数据）。
